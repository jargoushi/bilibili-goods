# 06 脚本库

## 1. 模块定位

把对标脚本拆成可复用组件，实现“按组件拼装脚本”，减少从零写稿。

对应代码目录：
- `app/modules/script_library/parser.py`
- `app/modules/script_library/library.py`

## 2. 组件模型

- `hook`：开头钩子
- `turn`：转折句
- `point`：卖点表达
- `cta`：结尾引导

每条组件都绑定标签：人群、场景、需求、语气、时长。

## 3. 输入与输出

- 输入：字幕文本 / 人工脚本
- 输出：结构化组件列表 + 标签 + 质量分

## 4. 核心流程

1. 文本分段（句级别/语义级别）。
2. 规则 + LLM 双阶段分类到组件类型。
3. 人工校正入口（可改类型与内容）。
4. 入库并建立检索索引。

## 5. 接口设计

- `POST /api/script-library/parse`
- `GET /api/script-library/search?scene=宿舍&crowd=学生`
- `POST /api/script-library/component`

## 6. 质量控制

- 组件长度约束（防止过长难复用）。
- 去重策略（文本相似度阈值）。
- 低质量组件打回待审核池。

## 7. 监控指标

- 新增组件数
- 复用率
- 人工改动率
- 可用组件覆盖率（场景 x 人群）

## 8. MVP 验收标准

- 可从 20 条对标视频中沉淀 >= 200 条组件。
- 搜索结果可直接用于脚本生成器拼装。
