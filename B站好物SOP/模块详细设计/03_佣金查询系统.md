# 03 佣金查询系统

## 1. 模块定位

将商品链接转换为联盟可识别商品 ID，批量查询佣金比例和预估收益，为选品排序提供核心输入。

对应代码目录：
- `app/modules/commission/jd_union.py`
- `app/modules/commission/taobao_union.py`

## 2. 边界定义

- 负责：联盟 API 封装、批量查询、结果归一化。
- 不负责：平台链接提取和商品详情爬取。

## 3. 输入与输出

- 输入：`platform + product_id` 列表
- 输出：
- `commission_rate`
- `estimated_income`
- `campaign_name`
- `query_status`

## 4. 数据结构

- `products` 表关键字段：
- `platform`
- `product_id`
- `commission_rate`
- `estimated_income`
- `updated_at`

## 5. 核心流程

1. 读取待查询商品清单。
2. 按平台分组调用联盟 API。
3. 统一转换佣金单位和字段格式。
4. 写回 `products` 表并记录更新时间。
5. 返回按佣金排序结果。

## 6. 接口设计

- `POST /api/commission/query`
- 请求：`{"items":[{"platform":"jd","product_id":"123"}]}`
- 返回：每个商品的佣金信息和状态

- `GET /api/commission/top?limit=50`
- 返回高佣金候选池

## 7. 异常与重试

- 鉴权失效：返回明确错误码并提示更新密钥。
- 单商品查询失败：跳过继续，不阻塞批次。
- 接口限流：分片 + 指数退避。

## 8. 监控指标

- 查询成功率
- 平均查询时延
- 鉴权失败次数
- 查询成本（每 1000 次请求）

## 9. MVP 验收标准

- 可批量查询 >= 100 个商品。
- 结果可落库并用于排序。
- 查询失败商品可单独重试。
