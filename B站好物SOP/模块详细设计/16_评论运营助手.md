# 16 评论运营助手

## 1. 模块定位

围绕评论区转化，提供评论监控、回复建议、人工审核、数据看板和收益报表。

对应代码目录：
- `app/modules/operation/comment_monitor.py`
- `app/modules/operation/reply_advisor.py`
- `app/modules/operation/dashboard.py`
- `app/modules/operation/revenue_report.py`

## 2. 输入与输出

- 输入：视频评论流、商品知识库、历史回复数据
- 输出：
- 待处理评论队列
- 回复建议草稿
- 运营指标看板
- 收益统计报表

## 3. 评论分类

- 求推荐
- 问参数
- 比价
- 售后
- 闲聊

## 4. 核心流程

1. 定时抓取最新评论。
2. 自动分类并判定优先级。
3. 基于知识库生成回复建议。
4. 人工审核后发送。
5. 回写结果并统计转化。

## 5. 接口设计

- `GET /api/operation/comments?status=pending`
- `POST /api/operation/reply/suggest`
- `POST /api/operation/reply/approve`
- `GET /api/operation/dashboard`
- `GET /api/operation/revenue-report`

## 6. 风险与控制

- 禁止全自动发送回复，必须人工确认。
- 高风险问题（售后、价格争议）强制人工编写。
- 回复模板要保留引用依据，避免答非所问。

## 7. 监控指标

- 评论响应时长
- 回复采纳率
- 回复后转化率
- GMV / 佣金趋势

## 8. MVP 验收标准

- 新评论 5 分钟内入队。
- 回复建议可用率 >= 80%（人工评估）。
- 能按视频和时间段输出基础收益报表。
