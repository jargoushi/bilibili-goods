# 02 评论区商品提取器

## 1. 模块定位

从评论区抓取商品线索，提取并标准化链接，分类到京东/淘宝/天猫平台。

对应代码目录：
- `app/modules/link_extractor/comment_crawler.py`
- `app/modules/link_extractor/link_parser.py`

## 2. 边界定义

- 负责：评论抓取、置顶优先、短链还原、平台分类、去重。
- 不负责：佣金查询和详情页抓取。

## 3. 输入与输出

- 输入：`video_id` 或 `video_url`
- 输出：
- `comment_id`
- `raw_link`
- `resolved_link`
- `platform`
- `product_id`
- `extracted_at`

## 4. 数据结构

- `comment_links`（建议新增）：
- `video_id`
- `comment_id`
- `user_id`
- `raw_text`
- `resolved_link`
- `platform`
- `product_id`
- `is_top_comment`

## 5. 核心流程

1. 分页抓取评论，优先抓置顶评论。
2. 正则抽取 URL 与口令片段。
3. 对短链做 HEAD 跳转还原。
4. 按域名规则分类平台并解析商品 ID。
5. 结果去重后写入数据库。

## 6. 接口设计

- `POST /api/comments/extract`
- 请求：`{"video_id": 1}`
- 返回：`{"total_comments": 280, "total_links": 32, "valid_links": 20}`

- `GET /api/comments/links?video_id=1`
- 返回该视频下所有有效商品链接

## 7. 异常与重试

- 评论接口限流：降速 + 重试。
- 短链不可达：标记状态 `resolve_failed`，后续可补偿任务重跑。
- 平台无法识别：标记 `unknown` 并保留原始链接。

## 8. 监控指标

- 评论抓取覆盖率
- 链接提取命中率
- 平台识别准确率
- 短链还原成功率

## 9. MVP 验收标准

- 单视频评论链接提取成功率 >= 90%（人工抽样）。
- 输出商品链接可直接用于佣金查询。
- 同一评论重复抓取不产生重复链接记录。
