# 07 卖点标签系统

## 1. 模块定位

将商品、脚本、评论统一映射到同一标签空间，支持按“周期 x 群体 x 需求”组合检索。

对应代码目录：
- `app/modules/tag_system/tagger.py`

## 2. 标签体系

- 一级：`场景`（宿舍/上班/户外）
- 一级：`人群`（学生/打工人/宝妈）
- 一级：`需求`（省钱/效率/舒适）
- 二级：可按类目扩展细分标签

## 3. 输入与输出

- 输入：商品参数、卖点文本、脚本文本
- 输出：标签集合 + 置信度 + 打标来源（规则/AI/人工）

## 4. 核心流程

1. 规则词典先打粗标签。
2. AI 模型做补全和歧义消解。
3. 人工确认高价值标签。
4. 写入标签映射表，建立倒排索引。

## 5. 接口设计

- `POST /api/tags/auto-tag`
- `POST /api/tags/manual`
- `GET /api/tags/search?scene=宿舍&need=省钱`

## 6. 数据结构

- `tags`：标签字典
- `tag_bindings`：对象与标签绑定
- `tag_bindings` 字段：`entity_type`, `entity_id`, `tag_id`, `score`, `source`

## 7. 监控指标

- 自动打标准确率（抽样）
- 人工修正率
- 标签覆盖率
- 检索命中率

## 8. MVP 验收标准

- 商品和脚本都支持统一标签检索。
- 三维组合检索响应 < 500ms（千级数据）。
