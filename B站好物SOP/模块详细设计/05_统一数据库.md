# 05 统一数据库

## 1. 模块定位

统一承接全链路数据，替代散落在 Excel/Notion 的信息，保证后续筛选、评分、推荐和导出都基于同一数据源。

对应代码目录：
- `app/database.py`
- `app/modules/data_store/models.py`
- `app/modules/data_store/crud.py`
- `app/modules/data_store/export.py`

## 2. 设计原则

- 单一事实源：同一商品只保留一个主记录。
- 可追溯：每条记录能追溯来源视频和抓取时间。
- 可演化：字段升级时兼容历史数据。

## 3. 核心实体

- `videos`：对标视频主表
- `products`：商品主表
- `comment_links`：评论链接表
- `product_details`：详情参数表
- `scripts`：脚本资产表
- `scores`：评分结果表
- `operations`：运营数据表

## 4. 关键索引

- `videos.source_url` 唯一索引
- `products(platform, product_id)` 联合唯一索引
- `comment_links(video_id, resolved_link)` 去重索引
- `scores(product_id, version)` 版本索引

## 5. 接口设计

- `GET /api/products`
- `POST /api/products/batch-upsert`
- `GET /api/products/export?format=xlsx`
- `GET /api/videos/{id}/products`

## 6. 导出规范

- Sheet1：商品池（佣金、评分、标签）
- Sheet2：视频来源映射
- Sheet3：参数与卖点
- 文件名：`goods_pool_YYYYMMDD_HHMM.xlsx`

## 7. 异常与数据治理

- 事务提交失败：整体回滚并记录失败批次。
- 脏数据标记：`status=invalid`，不直接删除。
- 每日去重任务：按主键规则合并重复商品。

## 8. 监控指标

- 日增商品数
- 重复数据率
- 导出成功率
- 查询响应时延

## 9. MVP 验收标准

- 输入 3 个对标视频后，可导出“商品+佣金”Excel。
- 支持按平台、佣金、标签过滤。
- API 查询在 500ms 内返回（千级数据量）。
